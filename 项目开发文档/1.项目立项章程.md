## 1.2 项目背景与目标
### 商业目标
- 解决外贸卖家（速卖通/TikTok）在耗材品类中"选什么品、怎么定价"的核心决策问题
- 降低试错成本，将选品从"拍脑袋"升级为"数据驱动"
- 预期效果：将选品成功率从行业平均的 20% 提升至 60% 以上

### 解决的核心问题
1. 每天面对海量 SKU 不知道选哪个
2. 无法实时监控竞品价格变动
3. 1688 进货价波动导致利润测算失真
4. 耗材品类风险（液体、易碎、知识产权）难以提前识别

## 1.3 项目范围
### 包含（In Scope）
- Sorftime 数据接入（选品情报）
- 1688 价格 API 接入（比价预警）
- GLM-5.2 为主、多模型备用的 AI 分析引擎
- 每日 Web 看板（含选品推荐、风险预警、AI 日报）
- 周度选品迭代逻辑
- 风险规则引擎（墨水/3D 耗材/相纸）

### 不包含（Out of Scope）
- 自动下单/采购功能
- 多用户权限管理（v1.0 仅限单用户）
- 移动端 App（仅 Web 端）
- 历史数据迁移（系统从零开始）

## 1.6 成功指标（可量化）
| 指标 | 目标值 | 度量方式 |
|------|--------|---------|
| 选品成功率 | ≥60% | 上架后 30 天内有动销的选品占比 |
| 选品响应时长 | ≤30 秒 | 从输入品类到返回清单的 p95 |
| 日报生成准时率 | 100% | 每日 09:00 前生成的占比 |
| 比价预警准确率 | ≥95% | 预警触发且价格确有变动的占比 |
| 看板首页 LCP | ≤2.5 秒 | 前端性能监控 |

## 1.8 项目退出/搁置标准
- 若 Sorftime 或 1688 API 长期不可用且无替代数据源，触发搁置评审。
- 若选品成功率在上线 8 周后仍低于 35%，重新评估算法与数据源。
- 若月度 API 成本超出预算 200%，评审是否调整范围或引入缓存降本。
