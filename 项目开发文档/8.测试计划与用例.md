# 测试计划与用例

## 8.1 测试范围

| 模块 | 测试重点 |
|------|---------|
| 选品算法 | 阈值筛选逻辑、综合评分计算、推荐排序 |
| 比价监控 | 标题匹配准确率、价格变动计算、预警触发 |
| AI日报 | Prompt 构建、模型调用、输出格式、模型切换 |
| 风险引擎 | 规则匹配、标签生成 |
| Web看板 | 数据展示、筛选操作、配置保存 |
| 非功能 | 性能、数据一致性、容灾降级 |

## 8.2 测试环境

| 环境 | 用途 | 数据来源 |
|------|------|---------|
| dev（本地） | 开发自测 | mock fixture |
| staging | 集成/回归测试 | 真实 API（小额）+ mock |
| production | UAT 与上线 | 真实数据 |

- 三方外部 API 在测试中默认走 mock；staging 做少量真实联调验证鉴权链路。
- 测试数据库独立，不与生产共用；每次集成测试前用 Alembic 重建 schema + seed。

## 8.3 测试数据准备

- **Mock 数据集**：为 Sorftime/1688/GLM 三方接口准备本地 fixture（JSON），避免消耗免费额度。
- **边界数据**：构造恰好等于阈值、空结果、超大批量（1 万条）的数据。
- **故障注入**：模拟三方 API 超时、空响应、限流（429）。

## 8.4 核心功能用例

### TC-001：选品算法筛选（正常）
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 调用选品接口，传入品类 "3d_filament" | 返回产品列表 |
| 2 | 检查返回数据 | 所有产品均满足设定的阈值条件 |
| 3 | 检查排序 | 按综合评分降序排列 |
| 4 | 检查未通过产品 | 显示具体未通过原因（如"月销量低于5000"） |

### TC-002：选品算法边界
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 月销量恰好等于下限 5000 | 通过筛选 |
| 2 | 月销量为 4999 | 不通过，原因"月销量低于5000" |
| 3 | 品类无任何符合产品 | 返回空列表 + 空状态提示 |

### TC-003：1688标题匹配
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 输入标题："PLA 3D Printer Filament 1.75mm Black 1kg" | 系统自动提取关键词 |
| 2 | 调用 1688 搜索接口 | 返回 Top 3 匹配商品 |
| 3 | 检查匹配结果 | 首条结果应为 PLA 耗材相关 |
| 4 | 人工确认匹配 | 状态更新为 "confirmed" |

### TC-004：价格预警触发
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 模拟 1688 价格上涨 6% | 系统触发"成本预警" |
| 2 | 查看看板预警区 | 显示该产品及预警标签 |
| 3 | 模拟价格下跌 5% | 触发"降价利好" |
| 4 | 价格变动恰好 4.9% | 不触发预警 |

### TC-005：AI日报生成
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 调用日报生成接口 | 10 秒内返回结果 |

| 3 | 切换模型到 deepseek | 再次生成成功 |
| 4 | GLM-5.2 不可用（模拟） | 自动切换到 GLM-5.1，记录 model_used |

### TC-006：风险规则匹配
| 步骤 | 操作 | 预期结果 |

### TC-007：配置热生效

|------|------|---------|

| 步骤 | 操作 | 预期结果 |
|------|------|---------|

## 8.6 缺陷管理

- **分级**：P0 阻断（核心功能不可用）/ P1 严重 / P2 一般 / P3 轻微。
- **回归**：每轮修复后回归受影响用例 + 全量 P0 用例。
- **准入准出**：进入测试需冒烟通过；准出要求 0 个 P0、P1 ≤2 且均有规避方案。

## 8.7 测试周期与自动化

| 阶段 | 测试活动 | 自动化程度 |
|------|---------|-----------|
| 开发期 | 单元测试随提交运行 | 全自动（CI） |
| 联调期 | 集成测试 + mock 联调 | 半自动 |
| Phase 4 | 功能测试按用例执行 | 手动为主 |
| 上线前 | 回归 P0 + 性能 + 容灾 | 半自动 |
| 上线后 | 冒烟（每次发布）| 全自动 |

- CI 流水线：`lint → unit test (coverage≥80%) → build`，任一失败阻止合并。
- 性能与容灾用例（TC-008/009）在 staging 手动触发，结果记录进 UAT 报告。

- [ ] 所有风险规则正确匹配
